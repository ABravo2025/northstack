Northstack Current Process Flow

This page shows the current onboarding and dashboard flow for Northstack using a visual flowchart and key process notes.

Current public flow

Login page Public users can log in or register, and the current UI also exposes tenant creation.
Tenant creation Clicking Create New Tenant opens the tenant onboarding form and creates an owner account immediately.
Dashboard Once authenticated, users land on the dashboard with Employees and Clients tabs.
HR / Clients Employees and clients can be listed and created from the tenant-scoped dashboard.

Proposed controlled onboarding

Two branches Separate signup and login paths make the flow clearer for new and returning users.
Tenant creation channels Tenants can be created by self-service, by our internal team, or through an API.
Client creation channels Tenant users can create clients through the UI or via API.
Role distinction Company users and tenant users get different dashboards and permissions.
flowchart TD A[Public Access] --> B{Existing user?} B -->|Yes| C[Login Page] B -->|No| D[Signup Page] C --> E[POST /api/auth/login] D --> F[POST /api/auth/register] E --> G{Valid token?} F --> G G --> H[Dashboard] G --> I[Show auth error] H --> J{User role?} J -->|Company user| K[Admin portal] J -->|Tenant user| L[Tenant dashboard] K --> M[Create tenant via internal admin] K --> N[Create tenant via API] K --> O[Self-service tenant creation optional] L --> P[Create clients via UI] L --> Q[Create clients via API] L --> R[View tenant Employees Clients Payments]

Important observations

  • The current tenant creation path is public and self-service.
  • Tenant onboarding is coupled with owner account creation and immediate login.
  • This flow is good for open signup, but not ideal if you need admin-controlled onboarding.
